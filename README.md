# Hablot | Real Estate Marketplace App

Hablot is a full-stack, cross-platform mobile application that enables users to buy, sell, and explore premium real estate properties—including villas, apartments, houses, and studios. The platform features user authentication, dynamic property listings, search filters, and an interactive map interface to view property locations in real-time.

---

## 🚀 Tech Stack

| Technology | Purpose |
| :--- | :--- |
| **React Native** | Cross-platform mobile framework |
| **Expo (Router)** | Development workflow and file-based routing |
| **Clerk** | Secure user authentication and profile management |
| **Supabase** | PostgreSQL database, Realtime subscriptions, and Storage |
| **React Native Maps** | Interactive map view for tracking property locations |
| **Tailwind CSS (NativeWind)** | Utility-first styling for mobile interfaces |

---

## ✨ Features

* **User Authentication:** Secure sign-up, login, and OAuth (Google/Apple) via Clerk.
* **Property Listings:** Browse, search, and filter properties by category (Villa, Apartment, House, Studio), price, and location.
* **Interactive Maps:** View available properties on an integrated map with custom markers indicating price and details.
* **Buy & Sell Portals:** * **Buyers:** Save favorite properties, contact sellers directly, and view high-resolution image galleries.
    * **Sellers:** List new properties with detailed descriptions, images, pricing, and precise geographical coordinates.
* **Image Uploads:** Seamless media uploads for property listings stored securely in Supabase Buckets.

## 🛠️ Architecture & Database Schema

The backend is built on **Supabase (PostgreSQL)**. Below is the simplified relational schema utilized for this application:

### Profiles Table
 * id uuid default gen_random_uuid() primary key,
 * clerk_id text unique not null,
 * email text not null,
 * first_name text,
 * last_name text,
 * avatar_url text,
 * is_admin boolean default false,
 * created_at timestamp with time zone default now()

### Properties Table
 * id uuid default gen_random_uuid() primary key,
 * title text not null,
 * description text,
 * price numeric not null,
 * type text not null, -- 'apartment' | 'house' | 'villa' | 'studio'
 * bedrooms int not null default 1,
 * bathrooms int not null default 1,
 * area_sqft int,
 * address text not null,
 * city text not null,
 * latitude float,
 * longitude float,
 * images text[] default '{}', -- array of Supabase Storage URLs
 * is_featured boolean default false,
 * is_sold boolean default false,
 * created_at timestamp with time zone default now()

### Saved Property Table
* id uuid default gen_random_uuid() primary key,
*  user_clerk_id text not null references users(clerk_id) on delete cascade,
* property_id uuid not null references properties(id) on delete cascade,
*  created_at timestamp with time zone default now(),
* unique(user_clerk_id, property_id) -- prevents duplicate saves

## 📦 Getting Started

### Prerequisites

Ensure you have the following installed on your local machine:
* Node.js (v18 or higher)
* npm or bun
* Expo Go app on your physical device or an emulator configured (Android Studio / Xcode)

### Installation Steps

1.  **Clone the Repository**
    ```bash
    git clone [https://github.com/shreyavni/hablot.git](https://shreyavni/hablot.git)
    cd hablot
    ```

2.  **Install Dependencies**
    ```bash
    npm install
    # or
    bun install
    ```

3.  **Environment Variables Setup**
    Create a `.env` file in the root directory and populate it with your API keys:
    ```env
    EXPO_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_...
    EXPO_PUBLIC_SUPABASE_URL=[https://your-project.supabase.co](https://your-project.supabase.co)
    EXPO_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOi...
    ```

4.  **Start the Development Server**
    ```bash
    npx expo start
    ```
    Press `a` for Android Emulator, `i` for iOS Simulator, or scan the QR code using the Expo Go app on your phone.

## 🔒 Security & Policies

* **Authentication Validation:** Supabase Row Level Security (RLS) is tied with JWT tokens generated by Clerk to ensure users can only modify or delete their own listed properties.
* **Secure API Requests:** Environment configurations are properly managed to prevent accidental leakage of master keys.
